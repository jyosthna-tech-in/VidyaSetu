# Chapter 5: Computerised Accounting System
### NCERT Class 11 Accountancy Part 2

---

## Chapter Purpose

This chapter explains the nature of a computerised accounting system, how it differs from manual accounting, why organisations use it, what its limitations are, and how accounting software should be selected.

The central idea is simple:

> A computerised accounting system uses computers, accounting software, databases, and procedures to record financial transactions and produce accounting reports according to accounting principles.

The computer changes the method of processing. It does not change the basic logic of accounting.

After studying this chapter, you should be able to:

## 1. Meaning of Computerised Accounting System

A computerised accounting system is an accounting information system that processes financial transactions and events through computers to produce reports required by users.

It must follow accounting principles. It is not just a calculator or a typing tool. It is a system that:

1. Receives accounting data.
2. Stores the data in an organised way.
3. Processes the data according to accounting rules.
4. Produces ledgers, statements, and reports.
5. Helps users make decisions.

### Simple Example

When a cash purchase of goods is entered into accounting software:

- The purchase account is affected.
- The cash account is affected.
- Reports such as cash book, trial balance, and financial statements can reflect the effect.
- The transaction is stored in the accounting database.

The accountant still needs to know that the correct entry is related to purchase and cash. The software only processes what is entered according to its design.

---

## 2. Two Basic Requirements

Every computerised accounting system has two basic requirements:

| Requirement | Meaning |
|---|---|
| Accounting framework | The accounting principles, account codes, grouping rules, and classification structure used by the system |
| Operating procedure | The method by which the system is used in the organisation's actual environment |

### 2.1 Accounting Framework

The accounting framework tells the system how accounts are organised.

It includes:

- Accounting principles
- Chart of accounts
- Account groups
- Voucher types
- Coding structure
- Rules for classification
- Reporting structure

Example:

If rent paid is grouped under indirect expenses, the system can place it correctly in the Profit and Loss Account.

### 2.2 Operating Procedure

Operating procedure means the practical steps followed while using the system.

It includes:

- Who can enter vouchers
- Who can approve entries
- How data is backed up
- How errors are corrected
- How passwords are managed
- When reports are generated
- How year-end processing is done

The same software may work differently in two organisations because their operating procedures differ.

---

## 3. Database-Oriented Nature of Computerised Accounting

Modern computerised accounting systems are usually database-oriented.

A database is an organised collection of data. A Database Management System, or DBMS, manages and organises data and allows software to access it.

In accounting, the database stores transaction details such as:

- Date
- Voucher number
- Account code
- Party name
- Amount
- Narration
- Tax details
- Payment mode
- Inventory details, if applicable

The accounting software uses this database to generate ledgers, trial balance, reports, and statements.

---

## 4. Four Additional Requirements of Database-Based Accounting

A computerised accounting system has four important database-related requirements.

| Requirement | Meaning | Accounting example |
|---|---|---|
| Front-end interface | The screen or form through which the user interacts with the system | Purchase voucher entry screen |
| Back-end database | Hidden storage where data is stored and retrieved | Accounting database storing vouchers |
| Data processing | Steps that transform data into useful information | Posting, grouping, calculation, report preparation |
| Reporting system | Set of tools and formats used to produce reports | Ledger, trial balance, stock report, financial statements |

### 4.1 Front-End Interface

The front-end interface is what the user sees and uses.

Examples:

- Voucher entry form
- Menu screen
- Report selection screen
- Customer master form
- Inventory item screen

A good front-end should reduce mistakes by using proper fields, lists, validation, and clear labels.

### 4.2 Back-End Database

The back-end database stores the actual accounting data. Users may not see the database directly, but every voucher and report depends on it.

It should be:

- Organised
- Secure
- Accurate
- Accessible only to authorised users
- Properly backed up

### 4.3 Data Processing

Data processing transforms raw transaction data into information.

Examples:

| Data entered | Processing result |
|---|---|
| Sales voucher | Sales account and customer account updated |
| Salary voucher | Salary expense and cash/bank account updated |
| Purchase of goods | Purchase account, supplier account, and stock may be updated |

### 4.4 Reporting System

The reporting system converts processed data into reports.

Examples:

- Cash book
- Ledger
- Trial balance
- Profit and Loss Account
- Balance Sheet
- Debtors ageing report
- Stock valuation report

The report should match the user's need. A manager may need a summary; an accountant may need detailed ledger information.

## 5. Manual Accounting vs Computerised Accounting

Accounting steps remain conceptually the same, but the method changes.

### 5.1 Identification

| Manual accounting | Computerised accounting |
|---|---|
| Accountant identifies transactions using accounting principles. | Accountant still identifies transactions using accounting principles. |

Identification is common to both systems. The computer does not decide whether a transaction is capital or revenue unless the user and software rules classify it.

### 5.2 Recording

| Manual accounting | Computerised accounting |
|---|---|
| Transactions are recorded in books of original entry. | Transaction data is entered into voucher screens and stored in a database. |

### 5.3 Classification

| Manual accounting | Computerised accounting |
|---|---|
| Entries are posted to ledger accounts. | Stored transaction data is processed and shown as ledger reports. |

In manual accounting, posting is a separate visible step. In computerised accounting, ledger output is produced from stored transaction data.

### 5.4 Summarising

| Manual accounting | Computerised accounting |
|---|---|
| Ledger balances are used to prepare trial balance. | Software processes stored data and produces trial balance report. |

### 5.5 Adjusting Entries

| Manual accounting | Computerised accounting |
|---|---|
| Adjusting entries are passed manually. | Journal vouchers or adjustment vouchers are entered and stored. |

Computerised accounting still needs adjustment entries. The difference is that they are entered as vouchers and processed by the system.

### 5.6 Financial Statements

| Manual accounting | Computerised accounting |
|---|---|
| Financial statements are prepared after trial balance and adjustments. | Software can generate financial statements from stored transaction data. |

### 5.7 Closing the Books

| Manual accounting | Computerised accounting |
|---|---|
| Closing and reversing entries are posted manually. | Year-end processing creates opening balances for the next period. |

### Key Conclusion

> Manual and computerised accounting follow the same accounting principles. The difference lies in the method, speed, storage, processing, and reporting.

## 6. Advantages of Computerised Accounting

### 6.1 Speed

Computerised accounting processes data faster than manual accounting.

Example:

A trial balance can be generated immediately after vouchers are entered instead of manually totaling ledger balances.

### 6.2 Accuracy

Calculations are more accurate because the computer performs arithmetic operations automatically.

However, accuracy depends on:

- Correct data entry
- Proper software design
- Correct account selection
- Strong validation controls

### 6.3 Reliability

Computers handle repetitive tasks consistently.

Examples:

- Posting multiple vouchers
- Generating ledgers
- Calculating totals
- Preparing repeated monthly reports

### 6.4 Up-to-Date Information

Accounting records are updated as soon as valid data is entered.

Example:

When a cash sale is recorded, cash balance, sales account, and related reports can update immediately.

### 6.5 Real-Time User Interface

Users can access information on a real-time basis when systems are connected through networks.

Example:

Head office may view branch sales data quickly if branches are connected to the accounting system.

### 6.6 Automated Document Production

Computerised systems can automatically produce:

- Invoices
- Cash book
- Trial balance
- Ledgers
- Statements of account
- Purchase reports
- Sales reports

### 6.7 Scalability

Computerised systems can handle additional transactions with relatively small additional processing cost.

Example:

If monthly vouchers increase from 500 to 5,000, the system can still process them if hardware, software, and procedures are adequate.

### 6.8 Legibility

Computer-generated records are easier to read than handwritten records. This reduces confusion caused by unclear handwriting.

### 6.9 Efficiency

Computerised accounting saves time and resources. It allows employees to focus more on analysis, control, and decision support rather than repetitive calculations.

### 6.10 Quality Reports

Reports can be generated in standard formats with inbuilt checks.

Examples:

- Trial balance
- Stock valuation
- Debtors ageing
- GST report
- Payroll report
- Profit and Loss Account
- Balance Sheet

### 6.11 MIS Reports

Computerised accounting supports management information reports.

Examples:

| MIS report | Use |
|---|---|
| Debtors ageing report | Helps identify delayed collections |
| Stock status report | Helps control inventory |
| Cash position report | Helps plan payments |
| Sales analysis | Helps evaluate customers, products, or regions |

### 6.12 Storage and Retrieval

Electronic storage uses less physical space than manual records. Data can also be retrieved quickly.

Example:

A supplier ledger for a previous month can be searched and printed without manually checking many pages.

### 6.13 Motivation and Employee Interest

Employees may feel more skilled and valued when trained to use modern accounting systems. However, this advantage appears only when training and change management are handled properly.

## 7. Limitations of Computerised Accounting

Computerised accounting also has risks and limitations.

### 7.1 Cost of Training

Accounting staff may need training in:

- Software use
- Voucher entry
- Report generation
- Backup procedures
- Error correction
- Security controls

Training can be costly, especially when software is complex or frequently updated.

### 7.2 Staff Opposition

Employees may resist computerisation because they fear:

- Job loss
- Reduced importance
- Difficulty learning new systems
- Change in work habits

Organisations must manage this resistance through training and communication.

### 7.3 Disruption

When a business shifts from manual to computerised accounting, routine work may be disturbed temporarily.

Reasons:

- New procedures
- Data migration
- Staff training
- Software setup
- Adjustment of controls

### 7.4 System Failure

Hardware failure, software failure, power problems, or database damage can interrupt accounting work.

Protection measures:

- Regular backups
- Reliable hardware
- Antivirus protection
- Uninterrupted power supply
- Disaster recovery plan

### 7.5 Inability to Check Unanticipated Errors

Computers can detect only those errors for which checks are designed.

Example:

Software may reject a blank amount, but it may not detect that a valid amount was entered under the wrong account.

### 7.6 Breach of Security

Computerised records may be exposed to:

- Password misuse
- Unauthorised access
- Data alteration
- Hacking
- Fraud through manipulation of software or records

Security controls are therefore essential.

### 7.7 Ill Effects on Health

Long computer use may cause:

- Eye strain
- Back pain
- Muscular pain
- Fatigue

This affects working efficiency if ignored.

### Key Balance

Computerised accounting is efficient, but it requires controls. The system should be protected through passwords, backups, access rights, audit trails, and trained users.

## 8. Sourcing of Accounting Software

Sourcing means selecting and acquiring suitable accounting software for the organisation.

The need for accounting software arises mainly in two situations:

1. When an organisation shifts from manual accounting to computerised accounting.
2. When an existing computerised system must be replaced due to changed needs.

The main rule is:

> People remain responsible for accounting. Software is only a tool that supports accounting work.

---

## 9. Categories of Accounting Software

Accounting software can be classified into three categories:

| Category | Suitable for |
|---|---|
| Ready-to-use software | Small and conventional organisations |
| Customised software | Medium or large organisations with special needs |
| Tailored software | Large organisations with complex, unique, and integrated needs |

### 9.1 Ready-to-Use Software

Ready-to-use software is standard software available for general use.

Features:

- Low installation cost
- Easy to learn
- Suitable for small businesses
- Limited number of users
- Limited flexibility
- Lower secrecy and security compared to advanced systems
- Usually not deeply linked with other information systems

Suitable for:

- Small shop
- Small service business
- Small non-profit organisation
- Business with low transaction volume

Example situation:

A small stationery shop with one accountant and simple sales and purchase records may use ready-to-use software.

### 9.2 Customised Software

Customised software is standard software modified to meet special user requirements.

Features:

- Higher cost than ready-to-use software
- More flexible
- Suitable for medium and large businesses
- Can be linked to other systems
- Better secrecy and user control
- Requires more training

Suitable for:

- Medium business with special inventory needs
- Organisation needing customised reports
- Business with multiple users and special controls

Example situation:

A distributor wants sales vouchers to update customer accounts and inventory status in a specific way. Standard software may be modified for this requirement.

### 9.3 Tailored Software

Tailored software is specially developed for a specific organisation.

Features:

- High cost
- Designed for exact needs
- Suitable for large and complex businesses
- Supports multiple users
- Strong security and authenticity checks
- Can become an important part of organisational MIS
- Requires specialised training

Suitable for:

- Large manufacturing company
- Multi-branch enterprise
- Large organisation with integrated HR, production, inventory, sales, and accounting systems

Example situation:

A large company with factories, warehouses, branches, payroll, production, inventory, and finance systems may need tailored software.

## 11. Choosing the Right Software

Use this decision guide:

| Organisation situation | Suitable software |
|---|---|
| One office, one user, few transactions | Ready-to-use |
| Medium business needing special reports | Customised |
| Large business with many branches and integrated departments | Tailored |
| Low budget and simple accounting | Ready-to-use |
| High secrecy and many users | Customised or tailored |
| Need to connect accounting with production, HR, and inventory | Tailored |

## 12. Factors Before Sourcing Accounting Software

### 12.1 Flexibility

The software should allow flexible:

- Data entry
- Report design
- User switching
- Hardware use
- Operating system use
- Business growth

A rigid system may become unsuitable as the organisation changes.

### 12.2 Cost of Installation and Maintenance

The organisation must consider:

- Purchase price
- Installation cost
- Training cost
- Maintenance cost
- Update cost
- Cost of adding modules
- Data recovery cost

Cheap software may become expensive if maintenance and alteration costs are high.

### 12.3 Size of Organisation

The size and transaction volume influence software choice.

| Organisation size | Likely need |
|---|---|
| Small | Simple, single-user software |
| Medium | Custom reports and multiple users |
| Large | Integrated, secure, multi-user software |

### 12.4 Ease of Adaptation and Training

Software should be learnable by the intended users.

Important questions:

- Can accounting staff learn it easily?
- Does the vendor provide training?
- Is the interface clear?
- Will users resist the system?
- Does the organisation have technical support?

### 12.5 Utilities and MIS Reports

The software should produce reports needed by management.

Examples:

- Final accounts
- Cash flow statement
- Ratio analysis
- Cost records
- Stock report
- Debtors ageing
- Payroll reports

If advanced reports are needed, ready-to-use software may not be sufficient.

### 12.6 Expected Level of Secrecy

Accounting data is sensitive. Software should provide proper security.

Security features may include:

- Passwords
- User rights
- Access restrictions
- Audit trail
- Data backup
- Data encryption

Example:

Purchase department may access purchase vouchers, while payroll data may be restricted to authorised HR and finance users.

### 12.7 Exporting and Importing Data

Many organisations need to move data between systems.

Examples:

- Export ledger data to spreadsheet software.
- Import bank statement for reconciliation.
- Share inventory data with accounting.
- Link accounting software with MIS software.

The data transfer should be clean, accurate, and secure.

### 12.8 Vendor Reputation and Capability

The vendor matters because accounting software needs support.

Consider:

- How long the vendor has been in business
- Number of existing users
- Quality of support
- Availability of updates
- Training support
- Ability to solve problems
- Reliability of maintenance service

Poor vendor support can make even good software difficult to use.

## 13. Important Distinctions

### Manual Accounting vs Computerised Accounting

| Basis | Manual accounting | Computerised accounting |
|---|---|---|
| Recording | Books of original entry | Voucher screens and database |
| Posting | Manual posting to ledgers | Processed from stored data |
| Trial balance | Prepared from ledger balances | Generated by software |
| Speed | Slower | Faster |
| Storage | Physical books | Electronic database |
| Errors | Arithmetic and posting errors common | Input and software design errors possible |
| Reports | Prepared manually | Generated automatically |
| Retrieval | Time-consuming | Quick |

### Ready-to-Use vs Customised vs Tailored

| Need | Best category |
|---|---|
| Lowest cost | Ready-to-use |
| Special modifications | Customised |
| Organisation-specific design | Tailored |
| Small business | Ready-to-use |
| Medium business | Customised |
| Large complex business | Tailored |

### Front-End vs Back-End

| Term | Meaning |
|---|---|
| Front-end | User-facing part, such as voucher entry screen |
| Back-end | Hidden database where records are stored |

## 14. Common Mistakes

| Mistake | Correct understanding |
|---|---|
| Saying computerised accounting changes accounting principles | Principles remain the same; method changes. |
| Thinking software is responsible for accounting decisions | People remain responsible for correct accounting. |
| Ignoring operating procedure | Procedures are essential for control and reliability. |
| Assuming computerised systems have no errors | Wrong input, wrong account selection, and weak software can cause errors. |
| Confusing customised and tailored software | Customised modifies existing software; tailored is specially designed. |
| Selecting software only by price | Maintenance, training, security, reports, and vendor support also matter. |
| Treating database as visible screen | Database is the organised back-end storage; the screen is the front-end. |

## 15. Exam Answer Frames

### Frame 1: Define Computerised Accounting System

Use this structure:

1. Define the system.
2. Mention financial transactions and events.
3. Mention accounting principles.
4. Mention production of reports.
5. Add database/software reference.

Model opening:

> A computerised accounting system is an accounting information system that uses computers and accounting software to process financial transactions and events according to accounting principles and produce reports required by users.

### Frame 2: Compare Manual and Computerised Accounting

Use this structure:

1. State that accounting principles are common.
2. Compare recording, classification, summarising, financial statements, and closing.
3. Mention speed, storage, retrieval, and report generation.
4. Conclude that method differs, not accounting logic.

### Frame 3: Advantages of Computerised Accounting

Use this structure:

1. Start with speed and accuracy.
2. Add reliability, up-to-date information, and scalability.
3. Add report quality, MIS reports, storage, retrieval, and efficiency.
4. Give one business example.

### Frame 4: Limitations of Computerised Accounting

Use this structure:

1. Mention training cost and staff opposition.
2. Mention disruption during changeover.
3. Mention system failure and security risk.
4. Mention inability to check unanticipated errors.
5. Conclude with need for controls.

### Frame 5: Sourcing Accounting Software

Use this structure:

1. Define sourcing.
2. Name software categories.
3. Explain factors: flexibility, cost, size, training, reports, secrecy, data transfer, vendor.
4. Match software type to organisation size and needs.

## Final Takeaway

A computerised accounting system improves speed, reporting, storage, and decision support, but it must be built on sound accounting principles, proper procedures, secure databases, trained users, and suitable software. The best system is not the most expensive one; it is the one that fits the organisation's accounting needs, controls, users, and reporting requirements.
